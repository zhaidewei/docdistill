# Doc Distill UI/UX Issues List

## Purpose

This document lists only the current UI/UX problems in the website.

It does not include redesign proposals, implementation plans, or solution directions.

## Product-Level Issues

1. The product currently feels more like a content viewer than a complete learning workspace.
2. The main product surfaces exist, but they do not form a clearly guided learning loop.
3. The site does not consistently help users answer "what should I do next?"

## Homepage Issues

1. The homepage behaves primarily like a marketing or showcase page rather than a learning dashboard.
2. The homepage explains what the product is, but does not help users decide where to start.
3. The homepage does not provide a clear continuation path for returning users.
4. The homepage does not expose recent activity, learning progress, or pending review work.
5. The homepage emphasizes product description more than actionable learning entry points.

## Navigation Issues

1. The top-level navigation structure reflects content areas, but not user tasks.
2. The navigation does not clearly distinguish between reading, reviewing, exploring, and managing saved learning artifacts.
3. The labeling of major areas does not strongly communicate their roles in a learning workflow.
4. Navigation consistency across desktop and mobile is weak.

## Learning Flow Issues

1. The system supports reading cards, but does not strongly guide users into a structured learning path.
2. The interface does not clearly support the transition from discovering content to reading it deeply.
3. The product does not make it easy to resume from where a user previously stopped.
4. The next recommended action is often implicit rather than explicit.

## Desktop Card Reading Issues

1. The desktop reading experience is functional, but card browsing depth is limited.
2. The card list relies heavily on tag filtering and lacks stronger navigation affordances.
3. The browsing flow can become cumbersome as the number of cards grows.
4. Card selection is initialized from the URL, but ongoing selection state is not clearly treated as shareable or persistent browsing context.
5. The reading area is strong for consumption, but weaker for progression and navigation continuity.

## Card Discovery Issues

1. Card discovery currently depends too much on manual list scanning.
2. The browsing flow does not sufficiently help users narrow content based on different intents.
3. The interface does not clearly separate exploratory browsing from focused reading.
4. The current discovery surface is better for viewing what exists than for deciding what matters next.

## Annotation Issues

1. Users can record stars, notes, and questions, but these interactions feel lightweight and incomplete.
2. Saved annotations are easy to create but harder to manage as they accumulate.
3. The annotation system behaves more like local capture than like a durable knowledge workflow.
4. The current reading experience supports adding notes, but does not strongly support revisiting or working through them later.
5. The note and question experience is biased toward quick entry rather than long-term usability.

## Notes/Library Issues

1. The current Notes page behaves more like a saved-export screen than a true personal library.
2. Saved learning artifacts are grouped, but not framed as an ongoing workspace.
3. The page is useful for collecting output, but less useful for navigating back into learning.
4. The connection between saved notes and their originating reading context is weak at the UX level.
5. The Notes area supports review of saved items, but not a strong sense of triage, follow-up, or continuity.

## Review Flow Issues

1. The review experience is strong in isolation, but weakly integrated into the broader product structure.
2. Review is treated more like a viewport-specific mode than a clearly named task area.
3. The system does not consistently present review as a first-class user intention.
4. The relationship between reading mode and review mode is not explicit enough.
5. The product does not clearly communicate when a user should read versus when a user should review.

## Mobile Experience Issues

1. Mobile users are pushed into a much narrower experience than desktop users.
2. On mobile, major product capabilities are hidden behind a mode decision made by screen size rather than user intent.
3. Mobile supports fast review well, but broader exploration and management are less visible.
4. The mobile experience is optimized for card swiping more than for full learning workflow coverage.
5. Mobile information architecture feels thinner and less complete than desktop.

## Knowledge Graph Issues

1. The graph is valuable for exploration, but it is not yet strong enough as a decision-making surface.
2. The graph gives structure, but not enough context to help users confidently choose the next action.
3. Topic exploration is more discoverable than card-level decision support.
4. The graph interaction model assumes a level of comfort with visual exploration that some users may not have.
5. The transition from graph exploration into focused reading is present, but not especially rich or supportive.

## Workflow Continuity Issues

1. The product surfaces are individually useful, but transitions between them are not strong enough.
2. The experience from discover -> read -> annotate -> revisit -> review does not feel fully connected.
3. Different parts of the site feel related, but not always orchestrated into a single workflow.
4. Users can perform important tasks, but the product does not always help them move naturally from one task to the next.

## State and Progress Visibility Issues

1. Important user state exists, but is not surfaced clearly enough in the UI.
2. Progress is visible in some contexts, but not as a cohesive cross-product concept.
3. The product does not consistently expose what has been learned, what is pending, and what deserves attention.
4. User momentum is tracked locally in parts of the app, but not surfaced as an intentional experience layer.

## IA Clarity Issues

1. The current page structure reflects implementation boundaries more than user mental models.
2. The site structure does not clearly map to the user's core tasks.
3. Reading, review, exploration, and saved knowledge management are all present, but not expressed with equal clarity.
4. The current information architecture is serviceable, but not yet intuitive as a learning system.
